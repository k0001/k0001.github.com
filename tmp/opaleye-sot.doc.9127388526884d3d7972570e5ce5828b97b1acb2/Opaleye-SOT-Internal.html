Associated Types

type PgType a :: *

PgType a indicates the primitive PostgreSQL column type that will ultimately be used as the index to opaleye's Column. This could be a itself, in the case of primitive types such as PGInt4, or it could be something else.

The motivation for this is the same as for “newtype wrappers” in Haskell. Say in you have this newtype in Haskell:

newtype UserId = UserId Int32

Now UserId and Int32 are different types, even though they share the same underlying representation Int32. With PgTyped you can achieve something similar:

instance PgTyped UserId where
  type PgType UserId = PGInt4

With that in place, you won't be able to accidentally mistake Kol PGInt4 for Kol UserId values, yet you will be able to easily reuse most of the machinery available to Kol PGInt4, except:

  • You will need a QueryRunnerColumnDefault instance to fetch Kol UserId values from the database as UserId:
  instance QueryRunnerColumnDefault PGInt4 UserId

You might find queryRunnerColumnFromWrapped useful for simple cases like UserId.

Notice that the instance mentions PGInt4 directly, not our PgTyped UserId. This is fine, not much would be gained by making the difference between them at this point, as you still need to do the FromRow parsing and any inconsistencies will be uncovered there.

  • If you want to reuse the exising Num or Fractional instances for Kol PGInt4, you will need to explicitely ask for it by instantiating PgNum UserId and/or PgFractional UserId. Most likely you won't need this for cases such as UserId@, since you shouldn't be doing arithmetic with user identifiers anyway.

data Kol a

class PgPrimType p => ToKol a p where

Build a Kol.

You need to provide a ToKol instance for every Haskell type you plan to convert to its PostgreSQL representation as Kol.

A a default implementation of kol is available for Wrapped instances:

default kol :: (PgTyped b, PgType b ~ p, Wrapped a, ToKol (Unwrapped a) p) => a -> Kol b
kol = kol . view _Wrapped'

Implementation notice: This class overlaps in purpose with Opaleye's Constant. Technicaly, we don't really need to repeat those instances here: we could just rely on Opaleye's Constant. However, as of today, Opaleye's Constant provides some undesired support which we deliberately want to avoid here. Namely, we don't want to support converting Int to PGInt4. If this is fixed upstream, we might go back to relying on Constant if suitable. See https://github.com/tomjaguarpaw/haskell-opaleye/pull/110

data Col name wd rn pgType hsType

Column description.

This is only used as a promoted datatype expected to have kind Col Symbol WD RN * *.

  • name: Column name.
  • wd: Whether DEFAULT can be written to this column (WD) or not (W).
  • rn: Whether NULL might be read from this column (RN) or not (R).
  • pgType: Type of the column value used in Opaleye queries as index to Kol or Koln. This must be an instance of PgTyped.
  • hsType: Type of the column value used in Haskell outside Opaleye queries. Hint: don't use something like Maybe Bool here if you want to indicate that this is an optional Bool column. Instead, use Int here and RN in the rn field.

Notice that Col is very different from Kol and Koln: Kol and Koln are used at runtime for manipulating values stored in columns, Col is used to describe the properties of a column at compile time.
